NetMapServer — API Reference

Base URL: http://<host>:8092  ·  All bodies are application/json  ·  Dates are ISO 8601 or Unix epoch (float)

Authentication schemes

Public No authentication required.
Optional Bearer Response filtered to user's vehicles when token provided; full data otherwise.
X-API-Key / Bearer Requires either a valid X-API-Key header or a Bearer token. Non-admin users see only their vehicles.
X-API-Key Header X-API-Key: <key> — used by iOS app and tracker devices.
Bearer Header Authorization: Bearer <token> — obtained from /api/auth/login.
Admin Bearer token with role: "admin".

Tracker & Journeys

GPS tracker telemetry ingestion and journey retrieval. Trackers push data via POST /api/vehicle-events. The server resolves the vehicle from the IMEI and auto-manages journey IDs.

POST /api/vehicle-events X-API-Key Ingest tracker telemetry (single or array)

Accepts a single object or a JSON array. The server resolves vehicleID from the IMEI pairing and assigns a journeyID automatically (journey_start events create a new journey).

Full upsert. The tracker will receive this config piggybacked on its next POST /api/vehicle-events call if configVersion is stale.

Body (TrackerConfigDTO)

Same body shape as PUT but all fields are optional — only provided keys are updated. Returns 200 OK.

POST /api/admin/trackers/:imei/apply-profile/:profileID Admin Apply a config profile to a tracker

Copies all settings from the named profile into the tracker’s TrackerConfig, increments configVersion, and saves. The tracker will receive the new config on its next event push. Returns 200 OK or 404.
