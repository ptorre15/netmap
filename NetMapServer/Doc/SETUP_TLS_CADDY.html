Procédure d'installation

1

Ouvrir les ports 80 et 443 sur la box

Dans l'interface admin de votre routeur (généralement http://192.168.1.1), section NAT / Redirection de ports :

Port externeProtocoleIP locale du MacPort interne
80TCP(voir ci-dessous)80
443TCP(voir ci-dessous)443

Pour connaître l'IP locale du Mac :

ipconfig getifaddr en0
# Exemple de résultat : 192.168.1.42
⚠
Attribuer une IP locale fixe (bail DHCP statique) au Mac dans la config de la box, pour que la règle NAT reste valide après un redémarrage.
2

Installer Caddy via Homebrew

brew install caddy

Vérifier l'installation :

caddy version
# caddy v2.x.x ...
3

Vérifier le Caddyfile

Le fichier NetMapServer/Caddyfile est déjà configuré pour ce déploiement :

92-137-172-240.nip.io {
    reverse_proxy localhost:8092

    header {
        Strict-Transport-Security "max-age=31536000; includeSubDomains; preload"
        X-Content-Type-Options    "nosniff"
        X-Frame-Options           "DENY"
        Referrer-Policy           "strict-origin-when-cross-origin"
        -Server
    }

    encode gzip zstd

    log {
        output file /var/log/caddy/netmap_access.log
        format json
    }
}

Valider la syntaxe :

cd /Users/phil/Projects/hub/NetMapServer
caddy validate --config Caddyfile --adapter caddyfile
4

S'assurer que Vapor écoute sur 127.0.0.1

Le code configure.swift a déjà été mis à jour. Vapor bind désormais sur 127.0.0.1:8092 par défaut. Vérifier que le serveur Vapor est démarré :

# Tester la connectivité locale
curl -s http://127.0.0.1:8092/api/sensors/latest | head -c 200
⚠
Si Vapor n'est pas accessible sur 127.0.0.1:8092, Caddy renverra des erreurs 502. Démarrer Vapor en premier.
5

Lancer Caddy

ℹ
Caddy a besoin des ports 80 et 443. Sur macOS, les ports < 1024 nécessitent sudo.
cd /Users/phil/Projects/hub/NetMapServer
sudo caddy run --config Caddyfile --adapter caddyfile

Lors du premier démarrage, Caddy :

  • Contacte Let's Encrypt via le challenge HTTP-01 (port 80)
  • Obtient un certificat pour 92-137-172-240.nip.io
  • Active le HTTPS sur le port 443
  • Redirige automatiquement HTTP → HTTPS

Pour lancer en arrière-plan comme service système :

sudo caddy start --config /Users/phil/Projects/hub/NetMapServer/Caddyfile --adapter caddyfile
6

Tester le HTTPS

# Tester depuis une machine externe (ou depuis le Mac en substituant l'IP)
curl -sv https://92-137-172-240.nip.io/api/sensors/latest 2>&1 | grep -E "SSL|HTTP|issuer"

Résultat attendu :

* SSL connection using TLSv1.3 / ...
* Server certificate:
*  issuer: C=US; O=Let's Encrypt; CN=R11
*  subject: CN=92-137-172-240.nip.io
*  expire date: Jun  3 2026
< HTTP/2 200
✅
Si vous voyez HTTP/2 200 et l'issuer Let's Encrypt, le setup est complet et conforme RED art. 3(3)(d)(e)(f).
7

Démarrage automatique au boot macOS

Pour que Caddy redémarre automatiquement avec le Mac :

# Créer un LaunchDaemon (nécessite sudo)
sudo tee /Library/LaunchDaemons/org.netmap.caddy.plist <<'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>org.netmap.caddy</string>
  <key>ProgramArguments</key>
  <array>
    <string>/opt/homebrew/bin/caddy</string>
    <string>run</string>
    <string>--config</string>
    <string>/Users/phil/Projects/hub/NetMapServer/Caddyfile</string>
    <string>--adapter</string>
    <string>caddyfile</string>
  </array>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>/var/log/caddy/caddy.log</string>
  <key>StandardErrorPath</key>
  <string>/var/log/caddy/caddy.err</string>
</dict>
</plist>
EOF

sudo mkdir -p /var/log/caddy
sudo launchctl load /Library/LaunchDaemons/org.netmap.caddy.plist
ℹ
Caddy renouvelle automatiquement les certificats Let's Encrypt avant expiration (~30 jours avant). Aucun cron ni intervention manuelle nécessaire.
8

Mettre à jour les clients (tracker GPS + app iOS)

Remplacer l'URL du serveur dans :

  • GPS tracker : configuration du firmware — remplacer http://92.137.172.240:8092 par https://92-137-172-240.nip.io
  • App iOS NetMap : ServerSettingsView.swift ou la valeur par défaut dans NetMapServerClient.swift
// NetMapServerClient.swift — exemple
let baseURL = "https://92-137-172-240.nip.io"
⚠
Si le firmware du tracker GPS ne supporte pas SNI ou TLS, vérifier dans sa documentation. La plupart des trackers modernes supportent HTTPS. Tester avec curl -v https://92-137-172-240.nip.io/api/records depuis le tracker ou un équivalent.

Checklist de déploiement

  • Port 80 ouvert sur la box (NAT → IP locale du Mac)
  • Port 443 ouvert sur la box (NAT → IP locale du Mac)
  • IP locale du Mac fixée (bail DHCP statique)
  • brew install caddy exécuté
  • Vapor démarré et accessible sur 127.0.0.1:8092
  • caddy validate --config Caddyfile --adapter caddyfile → OK
  • sudo caddy start ... exécuté → certificat obtenu
  • Test HTTPS depuis l'extérieur → HTTP/2 200 + Let's Encrypt

  • URL serveur mise à jour dans le firmware du tracker GPS
  • URL serveur mise à jour dans l'app iOS

Dépannage fréquent

⚠
Caddy ne peut pas obtenir le certificat — Le challenge HTTP-01 échoue. Vérifier : (1) port 80 ouvert sur la box, (2) aucun autre service n'écoute sur le port 80 du Mac (sudo lsof -i :80), (3) pare-feu macOS désactivé ou avec règle entrante sur port 80.
⚠
502 Bad Gateway — Caddy fonctionne mais Vapor est arrêté ou écoute sur la mauvaise adresse. Vérifier : curl http://127.0.0.1:8092/. Si BIND_HOST est défini à 0.0.0.0 pour les tests, le remettre à 127.0.0.1.
⚠
IP publique a changé — Si l'IP change (coupure, reboot box), le domaine nip.io ne sera plus valide. Contacter le FAI pour confirmer que l'IP est bien fixe, ou envisager DuckDNS si l'IP n'est pas garantie fixe.
ℹ
Logs Caddy en temps réel — tail -f /var/log/caddy/caddy.log ou sudo launchctl log show org.netmap.caddy.
