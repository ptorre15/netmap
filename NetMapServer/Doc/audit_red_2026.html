Critique Absence totale de chiffrement en transit (HTTP) Art. 3(3)(d)(e)(f) ▾
🚨 Ce finding est bloquant pour toute conformité RED. Aucune des exigences de confidentialité ou d'intégrité ne peut être satisfaite sans transport chiffré.

Constat

Le serveur Vapor est configuré en HTTP pur sur le port 8092 avec hostname = "0.0.0.0". Aucune configuration TLS n'est présente dans configure.swift.

Les flux suivants transitent intégralement en clair :

  • GPS tracker → serveur : IMEI, coordonnées GPS, données TPMS (réseau cellulaire)
  • App iOS → serveur : token Bearer, données capteurs BLE
  • Navigateur → serveur : mot de passe à la connexion, token Bearer
  • Serveur → navigateur : historique complet de géolocalisation

Un attaquant en position MITM (réseau Wi-Fi public, point d'accès rogue, interception cellulaire) peut capturer l'intégralité des données sans effort.

Recommandation — Reverse proxy Caddy (Let's Encrypt automatique)

# Caddyfile — renouvellement automatique, zéro configuration TLS
netmap.votredomaine.com {
    reverse_proxy localhost:8092
    header Strict-Transport-Security "max-age=31536000; includeSubDomains"
}

Recommandation — Lier Vapor à localhost uniquement

// configure.swift — après mise en place du reverse proxy
app.http.server.configuration.hostname = "127.0.0.1"
app.http.server.configuration.port     = 8092

Alternative — TLS natif Vapor (sans reverse proxy)

// Package.swift — ajouter la dépendance NIO SSL
.package(url: "https://github.com/apple/swift-nio-ssl.git", from: "2.0.0")

// configure.swift
import NIOSSL
let tlsConfig = TLSConfiguration.makeServerConfiguration(
    certificateChain: try NIOSSLCertificate.fromPEMFile("/etc/ssl/certs/cert.pem").map { .certificate($0) },
    privateKey: .file("/etc/ssl/private/key.pem")
)
app.http.server.configuration.tlsConfiguration = tlsConfig

Les endpoints suivants dans RecordController.swift n'exigent aucune authentification :

Critique Coordonnées GPS transmises à un serveur OSRM public tiers Art. 3(3)(e) ▾
🚨 Transfert de données personnelles (localisation) vers une tierce partie non identifiée sans base légale explicite — violation directe du RGPD article 28 et RED article 3(3)(e).

Constat

Le code JavaScript dans app.js appelle https://router.project-osrm.org pour le map-matching des trajets, en transmettant les coordonnées GPS brutes de chaque trajet. Ce serveur est public, opéré par une communauté open-source, sans DPA (Data Processing Agreement) possible.

Option A — Auto-hébergement OSRM (recommandé)

# Télécharger la carte de la région (ex. Île-de-France)
wget https://download.geofabrik.de/europe/france/ile-de-france-latest.osm.pbf

# Préparer les données OSRM
docker run -t -v $(pwd):/data osrm/osrm-backend \
  osrm-extract -p /opt/car.lua /data/ile-de-france-latest.osm.pbf
docker run -t -v $(pwd):/data osrm/osrm-backend osrm-partition /data/ile-de-france-latest.osrm
docker run -t -v $(pwd):/data osrm/osrm-backend osrm-customize /data/ile-de-france-latest.osrm

# Lancer le service
docker run -d -p 5000:5000 -v $(pwd):/data osrm/osrm-backend \
  osrm-routed --algorithm mld /data/ile-de-france-latest.osrm
// app.js — pointer vers le serveur local
const OSRM_BASE = "http://localhost:5000"  // au lieu de router.project-osrm.org

Option B — Désactiver le map-matching

Supprimer l'appel OSRM et afficher la trace GPS brute sur la carte. Solution immédiate sans infrastructure supplémentaire.

Option C — Valhalla auto-hébergé

Alternative plus légère pour les cartes régionales, avec support du routing matriciel et des isochrones.

Haute Aucune protection brute-force sur /api/auth/login Art. 3(3)(f) ▾

Constat

L'endpoint POST /api/auth/login dans AuthController.swift ne limite pas le nombre de tentatives d'authentification. Aucun compteur de tentatives, aucun délai, aucun verrouillage de compte. Un attaquant peut tester des millions de combinaisons email/mot de passe sans friction.

Recommandation — Middleware anti brute-force

// Sources/App/Middleware/BruteForceMiddleware.swift
import Vapor

actor BruteForceStore {
    var attempts: [String: (count: Int, resetAt: Date)] = [:]

    func check(ip: String, maxAttempts: Int = 5, windowSeconds: Double = 900) throws {
        let now = Date()
        if let e = attempts[ip], now < e.resetAt {
            if e.count >= maxAttempts {
                let retry = Int(e.resetAt.timeIntervalSince(now))
                throw Abort(.tooManyRequests,
                    reason: "Trop de tentatives. Réessayez dans \(retry)s",
                    headers: ["Retry-After": "\(retry)"])
            }
            attempts[ip] = (e.count + 1, e.resetAt)
        } else {
            attempts[ip] = (1, now.addingTimeInterval(windowSeconds))
        }
    }

    func reset(ip: String) { attempts.removeValue(forKey: ip) }
}

struct BruteForceMiddleware: AsyncMiddleware {
    static let store = BruteForceStore()

    func respond(to request: Request, chainingTo next: AsyncResponder) async throws -> Response {
        let ip = request.headers["CF-Connecting-IP"].first
               ?? request.remoteAddress?.ipAddress ?? "unknown"
        try await Self.store.check(ip: ip)
        let response = try await next.respond(to: request)
        // Succès → réinitialiser le compteur
        if response.status == .ok { await Self.store.reset(ip: ip) }
        return response
    }
}

// routes.swift — appliquer uniquement sur la route login
let loginRoute = app.grouped(BruteForceMiddleware())
loginRoute.post("api", "auth", "login", use: authController.login)

Haute Token Bearer stocké en localStorage (vulnérable XSS) Art. 3(3)(e)(f) ▾

Constat

Le token d'authentification est stocké dans localStorage. Tout script JavaScript injecté par XSS (CDN compromise, contenu tiers, injection) peut le lire et l'exfiltrer instantanément. Sans TLS (F1), le token est également visible en clair sur le réseau.

Recommandation — Migrer vers un cookie httpOnly

// AuthController.swift — réponse de connexion
func login(req: Request) async throws -> Response {
    // ... validation identifiants ...
    let token = try await createToken(for: user, on: req.db)

    let response = Response(status: .ok)
    // Sérialiser les infos non-sensibles dans le body JSON
    response.body = try .init(data: JSONEncoder().encode(LoginResponse(
        userID: user.id!, email: user.email, role: user.role
    )))
    response.headers.contentType = .json

    // Token dans un cookie inaccessible à JavaScript
    response.cookies["netmap_token"] = HTTPCookies.Value(
        string: token.value,
        expires: token.expiresAt,
        maxAge: Int(token.expiresAt.timeIntervalSinceNow),
        isHTTPOnly: true,   // ← inaccessible via JS
        isSecure: true,     // ← HTTPS uniquement (nécessite F1)
        sameSite: .strict   // ← protection CSRF
    )
    return response
}

// BearerAuthMiddleware.swift — lire le cookie en priorité
let tokenValue = request.headers.bearerAuthorization?.token
               ?? request.cookies["netmap_token"]?.string
// app.js — supprimer le stockage localStorage
// Avant :
localStorage.setItem('netmap_token', data.token)
// Après :
// Le cookie est géré automatiquement par le navigateur
// Toutes les requêtes fetch incluent credentials: 'include'
const res = await fetch('/api/data', {
    credentials: 'include',  // ← envoyer le cookie httpOnly
    headers: { 'Content-Type': 'application/json' }
})

Haute Absence de CSP et de SRI sur les dépendances CDN Art. 3(3)(e)(f) ▾

Constat

Le fichier index.html charge 4 ressources depuis des CDN externes sans attribut integrity (SRI) :

<!-- Actuellement — aucune vérification d'intégrité -->
<script src="https://unpkg.com/leaflet@1.9.4/dist/leaflet.js"></script>
<script src="https://cdn.jsdelivr.net/npm/chart.js@4.4.2/dist/chart.umd.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/chartjs-adapter-date-fns@3.0.0/..."></script>
<link href="https://fonts.googleapis.com/css2?family=Roboto..." rel="stylesheet">

Si un CDN est compromis (incident de supply chain), du code malveillant est exécuté dans le navigateur avec accès complet à la page. Absence de Content-Security-Policy pour limiter les sources autorisées.

Recommandation A — Ajouter les attributs SRI

<!-- Générer les hashes : curl -s URL | openssl dgst -sha384 -binary | base64 -->
<script
  src="https://unpkg.com/leaflet@1.9.4/dist/leaflet.js"
  integrity="sha384-[HASH_SHA384]"
  crossorigin="anonymous"></script>

Recommandation B — Content-Security-Policy dans Vapor

// Middleware/SecurityHeadersMiddleware.swift
struct SecurityHeadersMiddleware: AsyncMiddleware {
    func respond(to request: Request, chainingTo next: AsyncResponder) async throws -> Response {
        let response = try await next.respond(to: request)
        guard request.url.path.hasSuffix(".html") ||
              !request.url.path.contains(".") else { return response }

        response.headers.replaceOrAdd(name: "Content-Security-Policy", value: """
            default-src 'self';
            script-src 'self' https://unpkg.com https://cdn.jsdelivr.net;
            style-src 'self' https://fonts.googleapis.com 'unsafe-inline';
            font-src https://fonts.gstatic.com;
            img-src 'self' https://*.tile.openstreetmap.org data: blob:;
            connect-src 'self';
            frame-ancestors 'none';
        """)
        response.headers.replaceOrAdd(name: "X-Frame-Options", value: "DENY")
        response.headers.replaceOrAdd(name: "X-Content-Type-Options", value: "nosniff")
        response.headers.replaceOrAdd(name: "Referrer-Policy", value: "strict-origin-when-cross-origin")
        return response
    }
}

Moyenne Clé API unique partagée pour tous les appareils Art. 3(3)(d)(f) ▾

Constat

Tous les GPS trackers utilisent la même clé API (valeur dans app_settings ou fallback "netmap-dev"). Si un appareil est compromis ou volé, la révocation touche tous les appareils simultanément. Aucune traçabilité par appareil n'est possible.

Recommandation — Table des clés par appareil

// Models/DeviceKey.swift
final class DeviceKey: Model, @unchecked Sendable {
    static let schema = "device_keys"

    @ID(key: .id)               var id:         UUID?
    @Field(key: "value")        var value:      String  // 64 octets hex aléatoires
    @Field(key: "imei")         var imei:       String?
    @Field(key: "label")        var label:      String  // "Tracker camion 01"
    @Field(key: "is_active")    var isActive:   Bool
    @Field(key: "last_used_at") var lastUsedAt: Date?
    @Timestamp(key: "created_at", on: .create) var createdAt: Date?

    init() {}
}

// Middleware/APIKeyMiddleware.swift — requêter la table au lieu du singleton
struct APIKeyMiddleware: AsyncMiddleware {
    func respond(to request: Request, chainingTo next: AsyncResponder) async throws -> Response {
        guard let key = request.headers.first(name: "X-API-Key"),
              let _ = try await DeviceKey.query(on: request.db)
                  .filter(\.$value == key)
                  .filter(\.$isActive == true)
                  .first()
        else { throw Abort(.unauthorized, reason: "API key invalide ou révoquée") }

        // Mettre à jour last_used_at
        // ...
        return try await next.respond(to: request)
    }
}

Moyenne Base de données SQLite non chiffrée au repos Art. 3(3)(e) ▾

Constat

La base SQLite est stockée en clair à netmap_data.db. Elle contient :

  • Coordonnées GPS horodatées (données de localisation personnelle)
  • IMEI du tracker cellulaire (identifiant d'équipement unique)
  • Données TPMS (pression, température — proxies de comportement de conduite)
  • Hachages de mots de passe utilisateurs

En cas d'accès physique ou de fuite de backup, toutes ces données sont lisibles sans effort.

Recommandations

  • Niveau OS immédiat : activer FileVault sur macOS (System Preferences → Security → FileVault)
  • Niveau applicatif : migrer vers SQLCipher (AES-256 transparent)
  • Backups : chiffrer avec age ou GPG avant tout transfert distant
# Chiffrer un backup existant avec age
age -r "ssh-ed25519 AAAA..." netmap_data.db > netmap_data.db.age

Moyenne Scan BLE permanent avec allowDuplicatesKey: true Art. 3(3)(e) ▾

Constat

Dans BLEScanner.swift, le scan CoreBluetooth est lancé avec CBCentralManagerScanOptionAllowDuplicatesKey: true. Ce flag traite chaque advertisement reçu sans dédoublonnage, augmentant :

  • La surface de fingerprinting BLE (flux continu d'observations par MAC address)
  • La consommation énergétique (batterie iPhone)
  • Le volume de données inutilement traitées et potentiellement loggées

Note : les trames TPMS broadcast (Michelin, Steelmate) sont non chiffrées par conception du fabricant — ce risque résiduel est hors du périmètre de contrôle de l'application et doit être documenté dans la déclaration de conformité RED.

Recommandation

// BLEScanner.swift
centralManager.scanForPeripherals(
    withServices: nil,
    options: [
        CBCentralManagerScanOptionAllowDuplicatesKey: false  // ← désactiver
    ]
)
// Implémenter un throttle applicatif si mise à jour fréquente nécessaire :
private var lastSeenByPeripheral: [UUID: Date] = [:]
private let throttleInterval: TimeInterval = 2.0

func centralManager(_ central: CBCentralManager,
                    didDiscover peripheral: CBPeripheral,
                    advertisementData: [String: Any],
                    rssi RSSI: NSNumber) {
    let now = Date()
    if let last = lastSeenByPeripheral[peripheral.identifier],
       now.timeIntervalSince(last) < throttleInterval { return }
    lastSeenByPeripheral[peripheral.identifier] = now
    // ... traitement normal ...
}

Faible Politique de mots de passe insuffisante Art. 3(3)(f) ▾

Constat

Minimum de 6 caractères dans AuthController.swift. Mots de passe temporaires au format "Kx3-Bm7-Rp2" (~42 bits d'entropie). bcrypt sans vérification du facteur de coût configuré.

Recommandation

// AuthController.swift
func validatePassword(_ password: String) throws {
    guard password.count >= 12 else {
        throw Abort(.badRequest, reason: "Minimum 12 caractères requis")
    }
    guard password.contains(where: \.isUppercase) else {
        throw Abort(.badRequest, reason: "Au moins une majuscule requise")
    }
    guard password.contains(where: \.isNumber) else {
        throw Abort(.badRequest, reason: "Au moins un chiffre requis")
    }
    let specials = "!@#$%^&*()_+-=[]{}|;':\",./<>?"
    guard password.contains(where: { specials.contains($0) }) else {
        throw Abort(.badRequest, reason: "Au moins un caractère spécial requis")
    }
}

// Mots de passe temporaires — 128 bits d'entropie minimum
func generateTemporaryPassword() -> String {
    var bytes = [UInt8](repeating: 0, count: 16)
    _ = SecRandomCopyBytes(kSecRandomDefault, bytes.count, &bytes)
    return bytes.map { String(format: "%02x", $0) }.joined()
}

Faible Absence de journal d'audit des accès Art. 3(3)(e)(f) ▾

Constat

Aucun événement de sécurité n'est consigné : tentatives de connexion échouées, révocations de token, changements de clé API, accès aux données de géolocalisation. L'absence de logs rend toute investigation post-incident impossible.

// Models/SecurityEvent.swift
final class SecurityEvent: Model, @unchecked Sendable {

    @ID(key: .id)              var id:         UUID?

Faible Serveur exposé sur toutes les interfaces réseau Art. 3(3)(d) ▾

Constat

hostname = "0.0.0.0" dans configure.swift expose le service Vapor sur toutes les interfaces réseau de la machine (Wi-Fi, Ethernet, VPN, loopback). Sans TLS ni firewall applicatif, toute machine sur un réseau accessible peut atteindre directement le serveur applicatif.

Recommandation

// configure.swift — après mise en place du reverse proxy (F1)
// Lier uniquement à l'interface locale
app.http.server.configuration.hostname = "127.0.0.1"
app.http.server.configuration.port     = 8092
// Le reverse proxy (Caddy/nginx) est la seule porte d'entrée publique
⚠ Ce finding est conditionnel au déploiement d'un reverse proxy (F1). En l'absence de reverse proxy, garder 0.0.0.0 pour maintenir l'accessibilité, mais mettre TLS en priorité absolue.
