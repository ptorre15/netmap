NetMapServer — API Reference

Base URL: http://<host>:8092  ·  All bodies are application/json  ·  Dates are ISO 8601 or Unix epoch (float)

Authentication schemes

Public No authentication required.
Optional Bearer Response filtered to user's vehicles when token provided; full data otherwise.
X-API-Key Header X-API-Key: <key> — used by iOS app and tracker devices.
Bearer Header Authorization: Bearer <token> — obtained from /api/auth/login.
Admin Bearer token with role: "admin".

General

Health and liveness endpoints.

GET /health Public Server liveness probe

Returns a JSON object confirming the server is running. Used by monitoring and deployment scripts.

Response 200
{ "status": "ok", "server": "NetMapServer", "version": "1.0.0" }

Authentication

Session management for the web dashboard.

POST /api/auth/login Public Obtain a Bearer token

Authenticates with email + password and returns a session token.

Request body
FieldTypeReq?Description
emailstring✓User email address
passwordstring✓User password
Response 200
{ "token": "abc123…", "email": "user@example.com", "displayName": "Alice", "role": "admin" // "admin" | "user" }
POST /api/auth/logout Bearer Invalidate current token

Deletes the current session token. No body required. Returns 204 No Content.

POST /api/auth/setup Public (first-run only) Create the initial admin account

Only works when no users exist in the database. Creates the first admin and returns a Bearer token.

Request body
{ "email": "admin@example.com", "password": "secret" }

Sensors & Records

BLE sensor readings ingested from the iOS app (TMS, STIHL, ELA, AirTag, Tracker). All timestamps may be ISO 8601 or a Unix epoch float.

GET /api/sensors/latest Optional Bearer One summary row per sensor

Returns the most recent reading for every known sensor. Non-admin authenticated users only see sensors linked to their vehicles.

Response 200 — array of SensorStat
[{ "sensorID": "1B71F09D-…", "vehicleID": "UUID", "vehicleName": "Truck #1", "assetTypeID": "vehicle", "brand": "michelin", // michelin | stihl | ela | airtag | tracker "wheelPosition": "FL", // nullable "latestPressureBar": 2.45, "latestTemperatureC": 24.3, "latestVbattVolts": 2.94, "targetPressureBar": 2.5, "latestBatteryPct": 88, "latestChargeState": "Idle", "sensorName": "Front Left", "latestHealthPct": 95, // Stihl Smart Battery "latestChargingCycles": 12, "latestProductVariant": "coin", // ELA beacon type "latestTotalSeconds": 3600, "latestGpsSatellites": 8, "latestLatitude": 45.999, "latestLongitude": 6.122, "latestTimestamp": "2026-03-03T19:46:29Z", "readingCount": 427 }]
GET /api/sensors/:sensorID/puncture-risk Public Slow puncture detection

Analyses the last 60 pressure readings for a statistically significant downward trend. Requires ≥ 10 readings. Risk threshold: >0.08 bar drop AND >3% drop.

Response 200
{ "sensorID": "1B71F09D-…", "hasRisk": true, "pressureDrop": 0.12, // baseline − recent (bar) "baseline": 2.50, "recent": 2.38, "readingCount": 60, "message": "Pressure dropped 0.12 bar (4.8%)…" }
POST /api/sensors/pair X-API-Key Register a sensor pairing immediately

Creates or replaces a sensor↔vehicle link without requiring a live BLE reading. The sensor appears in /api/sensors/latest immediately.

Request body

Vehicles / Assets

Assets are auto-created from sensor payloads. These endpoints allow manual management. /api/assets is an alias for /api/vehicles.

GET /api/vehicles   /api/assets Optional Bearer List all vehicles / assets

Returns all assets. Non-admin authenticated users only see their linked assets.

Response 200 — array of Vehicle
[{ "id": "UUID", "name": "Truck #1", "assetTypeID": "vehicle", "createdBy": "system", "createdAt": "2026-01-15T08:00:00Z" }]
GET /api/vehicles/:id   /api/assets/:id Public Fetch one vehicle

Returns the Vehicle object for the given UUID or 404 if not found.

POST /api/vehicles Admin Create a vehicle / asset
FieldTypeReq?Description
namestring✓Asset display name
assetTypeIDstringoptDefault: "vehicle"

Returns the created Vehicle with 201 Created.

PATCH /api/vehicles/:id Admin Rename / retype a vehicle
FieldTypeDescription
namestringNew display name
assetTypeIDstringNew asset type

Returns 200 OK.

DELETE /api/vehicles/:id Admin Delete a vehicle and all its sensor data

Permanently deletes the vehicle and cascades to all linked sensor_readings. Returns 204 No Content.

Tracker & Journeys

GPS tracker telemetry ingestion and journey retrieval. Trackers push data via POST /api/vehicle-events. The server resolves the vehicle from the IMEI and auto-manages journey IDs.

POST /api/vehicle-events X-API-Key Ingest tracker telemetry (single or array)

Accepts a single object or a JSON array. The server resolves vehicleID from the IMEI pairing and assigns a journeyID automatically (journey_start events create a new journey).

Note: journeyDistanceKm is expected in metres (as sent by common GPS trackers). The server divides by 1000 for display.
Request body — VehicleEventPayload
FieldTypeReq?Description
imeistring✓Tracker IMEI
eventTypestringoptjourney_start · driving · journey_end (default: driving)
timestampISO 8601 / epochoptDefault: server time
latitude / longitudenumberoptGPS position
headingDegnumberoptBearing in degrees
speedKmhnumberoptSpeed in km/h
odometerKmnumberoptTotal odometer reading in km
distanceElapsedKmnumberoptDistance since last event (km)
journeyDistanceKmnumberoptCumulative journey distance (metres from tracker, stored as-is)
fuelLevelPctintoptFuel level 0–100 %
fuelConsumedLnumberoptFuel consumed this event (L)
journeyFuelConsumedLnumberoptCumulative journey fuel (L)
engineRpmintoptEngine RPM
engineTempCnumberoptEngine temperature (°C)
driverIDstringoptDriver identifier
obdCodestringoptOBD fault code

Returns 201 Created.

GET /api/vehicle-events Public List raw tracker events
Query parameters
ParamTypeDefaultDescription
journeystring (UUID)–Filter by journey ID
vehiclestring (UUID)–Filter by vehicle ID
imeistring–Filter by IMEI
event_typestring–Filter by event type
limitint1000Max events. Up to 2000 recommended for track display.

Returns array sorted ascending by timestamp.

GET /api/vehicle-events/journeys Public Journey summaries (one row per journey)

Permanently deletes all vehicle_events rows for the given journey UUID. Returns 204 No Content or 404 if not found.

Admin — API Key

All /api/admin/* routes require a Bearer token with role: "admin".

GET /api/admin/api-key Admin Retrieve the current API key
Response 200
{ "apiKey": "abc123…" }
POST /api/admin/api-key/rotate Admin Generate a new API key

Generates a new random API key, saves it in the database and returns it. The old key is immediately invalidated. Update all client configurations accordingly.

GET /api/admin/users Admin List all users
Response 200
[{ "id": "UUID", "email": "user@example.com", "displayName": "Alice", "role": "user", "createdAt": "2026-01-15T08:00:00Z", "assetIDs": ["UUID1", "UUID2"] }]
POST /api/admin/users Admin Create a new user
FieldTypeReq?Description
emailstring✓Email address (unique)
displayNamestringoptFull name
rolestringopt"admin" or "user" (default)

{ "id": "UUID", "email": "user@example.com", "role": "user", "password": "fleet247" // shown once, store securely }

FieldTypeDescription

DELETE /api/admin/users/:userID Admin Delete a user

Deletes the user, their tokens and all asset links. Returns 204. Cannot delete the last admin account.

Returns 204 No Content.

Admin — Tracker Management

GET /api/admin/trackers Admin List all registered trackers
Response 200
[{ "imei": "867280066365446", "vehicleID": "UUID", "vehicleName": "Truck #1", "sensorName": "Fleet Tracker", // nullable "readingCount": 1250 }]
POST /api/admin/trackers Admin Register a new tracker IMEI

Idempotent — if the IMEI already exists it updates the vehicle link.

FieldTypeReq?Description
imeistring✓Tracker IMEI
vehicleIDstring (UUID)✓Vehicle to link to
sensorNamestringoptHuman-readable label

Returns 201 Created.

PATCH /api/admin/trackers/:imei Admin Rename a tracker
{ "sensorName": "Fleet Tracker" }

Set sensorName to null to clear the label. Returns 200 OK.

POST /api/admin/trackers/pair Admin Reassign a tracker to a vehicle

Updates the vehicle link for both sensor_readings and all historical vehicle_events. Enforces 1 tracker per vehicle.

{ "imei": "867280066365446", "vehicleID": "UUID" }

Returns 200 OK or 409 Conflict if the vehicle already has a different tracker paired.

DELETE /api/admin/trackers/:imei/pair Admin Remove a tracker's vehicle link

Resets vehicleID to the IMEI itself (unlinked state) in sensor_readings and vehicle_events. Returns 204 No Content.

DELETE /api/admin/trackers/:imei Admin Delete all sensor_readings for a tracker

Removes all sensor_readings rows for the given IMEI. Does not delete vehicle_events history. Returns 204 No Content.
